# FE-002: Доска проекта — 5 колонок + Drag&Drop между ними

## Контекст
В SPRINT зафиксировано: доска проекта имеет 5 колонок (Queue / In Progress / Waiting / Done / Today), а перенос задач между колонками — **строго Drag&Drop** (без альтернативных кнопок как основного пути).

## Acceptance Criteria («Готово»)
- На странице проекта отображаются 5 колонок в заданном порядке:
  - Queue
  - In Progress
  - Waiting
  - Done
  - Today
- Карточки задач отображаются в своих колонках по `status`.
- Перенос карточки между колонками выполняется drag&drop:
  - в UI видно, куда будет помещена карточка (хотя бы подсветка колонки).
  - после drop фронт вызывает API для смены статуса.
  - при успешном ответе задача остаётся в новой колонке.
  - при ошибке API — откат в исходную колонку + показ ошибки.
- Минимальная анимация допустима, но drag&drop должен быть стабильным (без «залипания»).

## Как проверить
1. Открыть любой проект с несколькими задачами.
2. Перетащить задачу из Queue в In Progress → обновить страницу → задача должна остаться в новой колонке.
3. Сымитировать ошибку (например, выключить API/поломать токен) → попытка dnd должна показать ошибку и вернуть карточку.

## Риски / заметки
- Выбор библиотеки DnD: важно, чтобы работала в современных браузерах и не ломала скролл внутри колонок.
- Возможны гонки при быстром dnd нескольких карточек — нужен базовый лок/disabled на карточке во время сохранения.
