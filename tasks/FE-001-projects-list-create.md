# FE-001: Проекты — список + создание

## Контекст
Спринт требует: пользователь после Telegram логина видит список своих проектов и может создать проект (минимум: `name`). Сейчас во фронте заглушка с одним «Личным» проектом без сохранения.

## Acceptance Criteria («Готово»)
- [ ] После логина отображается список проектов из API (только проекты текущего пользователя).
- [ ] Есть UI для создания проекта:
  - [ ] обязательное поле: `name` (валидация на пустое значение).
  - [ ] по успешному созданию проект появляется в списке без перезагрузки.
- [ ] Ошибки API отображаются человеку (не молча в консоль).
- [ ] Роутинг/путь остаётся под `/taskManager/` (без поломки относительных ссылок).

## Статус по `STATUS.md`
- [x] FE реализовано (проекты/создание) — осталось дождаться финального BE-контракта/эндпоинтов при необходимости.

## Как проверить
1. Залогиниться через Telegram.
2. Открыть экран «Проекты» → убедиться, что список приходит с бэка.
3. Нажать «Создать проект», ввести имя, сохранить.
4. Убедиться, что проект появился в списке и повторный заход на страницу показывает его.

## Риски / заметки
- Нужно согласовать формат API (эндпоинты/поля) с BE, чтобы не плодить временные адаптеры.
- Учесть пустое состояние (нет проектов) — показать call-to-action.
