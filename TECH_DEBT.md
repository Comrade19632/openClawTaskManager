# TECH_DEBT (TaskManager)

Технический долг, найденный на code review / в процессе разработки.

- FE: почти весь UI/логика находится в одном файле `frontend/src/main.jsx` (включая fetch-клиент, экраны, компоненты). Это увеличит риск мердж-конфликтов и усложнит внедрение канбана/DnD. Рекомендуется разнести по модулям (`api/`, `features/`, `components/`).
- FE/BE контракт пока не зафиксирован (форматы статусов, board endpoint, ordering внутри колонок). Риск: переделка DnD и API после первой итерации.
- Ordering задач внутри колонки не определён: если в DnD потребуется сохранение порядка, нужно поле `position`/endpoint для reorder (лучше продумать заранее).
- Репозиторий содержит директорию `frontend/node_modules` в рабочем дереве (даже если не трекается git). Это может замедлять CI/сканы и увеличивать размер контекста сборки; желательно убедиться, что она не попадает в docker build context/артефакты.
- В репозитории присутствует файл `.env` (хотя он в `.gitignore`). Нужно удалить из git (и истории/коммитов, если уже попал) и хранить секреты только вне репо.
- Auth: используется DRF TokenAuth без срока жизни (для MVP ок), но в будущем может потребоваться ротация/expiry.
- Backend импортирует `django.contrib.auth.models.User` напрямую — если когда-нибудь перейдём на кастомную модель пользователя, придётся рефакторить (лучше использовать `get_user_model()`).
